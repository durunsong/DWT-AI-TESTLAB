# 页面断言与异常场景清单

本文档用于统一 Dowalet 核心流程的断言口径，避免自动化用例只完成点击操作而缺少业务结果校验。

## 通用断言

| 场景 | 建议断言 | 说明 |
| --- | --- | --- |
| 登录成功 | 首页标识、用户信息、URL 或关键菜单可见 | 当前已有 `user_home_marker`、`admin_home_marker`。 |
| 表单提交 | 成功提示、状态文本、提交按钮恢复可点击 | 必要时补充 `wait_for_api` 成功码。 |
| 列表查询 | 列表行、空态文案、分页或筛选条件回显 | 列表为空不是失败，但需要明确空态断言。 |
| 弹窗交互 | 弹窗标题、确认按钮、取消后弹窗关闭 | 涉及确认类操作建议保留截图。 |
| API 等待 | HTTP 状态、业务码、关键字段 | 使用 `business_code_path` 和 `success_codes`。 |
| 只读 DB 校验 | 关键状态、记录存在性、更新时间 | 仅用于测试环境，只读账号执行。 |

## P0 流程断言

| 流程 | 必选断言 | 建议异常场景 |
| --- | --- | --- |
| user 登录 | 登录后首页或工作台标识可见 | 密码错误、账号不存在、按钮禁用、验证码错误。 |
| admin 登录 | 后台首页或控制台标识可见 | 权限不足、密码错误、账号锁定。 |
| KYC 提交 | 提交接口成功、user 端状态为“审核中” | 必填项为空、附件格式错误、重复提交。 |
| KYC 审核 | admin 审核接口成功、user 端状态为“审核通过” | 审核拒绝、审核单不存在、重复审核。 |
| 钱包首页 | 资产模块、币种列表或关键入口可见 | 接口失败、空资产、敏感金额展示异常。 |
| 消息中心 | 未读数、消息列表、详情或空态可见 | 空列表、已读状态变化、接口失败。 |

## P1 流程断言

| 流程 | 必选断言 | 建议异常场景 |
| --- | --- | --- |
| 注册 | 注册成功提示、跳转登录或自动登录 | 重复账号、验证码错误、协议未勾选。 |
| 忘记密码 | 重置成功提示、可用新密码登录 | 弱密码、验证码过期、两次密码不一致。 |
| 手机号绑定 | 绑定成功提示、安全页手机号脱敏展示 | 手机号格式错误、验证码错误、已绑定手机号。 |
| 验证方式修改 | 修改成功提示、新方式生效 | 原验证失败、取消修改、频繁请求验证码。 |
| 后台资料修改 | 保存接口成功、页面提示“修改成功” | 密码错误、头像格式错误、字段超长。 |

## 失败产物要求

每条核心用例失败时应保留：

- 控制台运行日志。
- 步骤截图或失败截图。
- Playwright trace。
- HTML/JSON 报告。
- 可选视频，建议 `retain-on-failure`。

对外共享前需要检查报告、截图、trace、视频和上传资料是否包含真实账号、手机号、邮箱、证件号、客户名称、token 或生产域名。
